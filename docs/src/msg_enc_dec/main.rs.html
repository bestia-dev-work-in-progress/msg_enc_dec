86//! ## Cryptography
87//!
88//! Cryptography is a technique of securing information and communications using codes to ensure confidentiality, integrity and authentication.  
89//! Modern ciphers, such as the Advanced Encryption Standard (AES), are considered virtually unbreakable.  
90//! Secret key cryptography, also known as symmetric encryption, uses a single key to encrypt and decrypt a message.  
91//! Public key cryptography (PKC), or asymmetric cryptography, uses mathematical functions to create codes that are exceptionally difficult to crack. It enables people to communicate securely over a non-secure communications channel without the need for a secret key.  
92//! <https://www.fortinet.com/resources/cyberglossary/what-is-cryptography>
93//!
94//! ## OpenSSH
95//!
96//! OpenSSH is the premier connectivity tool for remote login with the SSH protocol. It encrypts all traffic to eliminate eavesdropping, connection hijacking, and other attacks.  
97//! Key management with ssh-add, ssh-keysign, ssh-keyscan, and ssh-keygen, ssh-agent.  
98//! <https://www.openssh.com/>
99//!
100//! Open SSH uses Ed22519 for authentication. The SSH servers has a list of public keys that are authorized. The handshake: The server sends a random message. The client signs it with the private key Ed25519. The SSH server verifies the signature with the public key Ed25519.
101//!
102//! OpenSSH comes with tools to manage keys and it is a knowledge every developer learns early and thoroughly. The private key is protected by a passphrase. For repetitive use of the same private key I can use ssh-agent to input the passphrase only once. Usually the key inside ssh-agent is time limited for example for one hour.
103//!
104//! ## bestia.dev
105//!
106//! I use Ed25519 to store encrypted values on the local disk. First I create random 32 bytes called the 'seed'. I sign it with the private key Ed25519. That becomes the password I use to symmetrically encrypt GCM the secret value. In the saved file there is in plain text the seed and the encrypted data. Only the owner of the private key Ed25519 can sign the seed to get the password to then decrypt GCM the data.
107//!
108//! ## Ed25519
109//!
110//! Ed25519 is the EdDSA signature scheme using SHA-512 (SHA-2) and an elliptic curve related to Curve25519.  
111//! In public-key cryptography, Edwards-curve Digital Signature Algorithm (EdDSA) is a digital signature scheme using a variant of Schnorr signature based on twisted Edwards curves. It is designed to be faster than existing digital signature schemes without sacrificing security.  
112//! Public keys are 256 bits long and signatures are 512 bits long.  
113//! <https://en.wikipedia.org/wiki/EdDSA#Ed25519>
114//!
115//! Ed25519 is a signature scheme. It does not do encryption.  
116//!
117//! ## X25519
118//!
119//! X25519 is the name given to the Elliptic Curve Diffie-Hellman (ECDH) key exchange built on Ed22519.  
120//! <https://medium.com/@aditrizky052/unlocking-the-power-of-curve25519-ed25519-x25519-the-modern-pillars-of-secure-and-high-speed-a3daefbad0a4>
121//!
122//! The Diffie-Hellman algorithm (DH) is used for secret key exchanges and requires two people to agree on a large prime number.  
123//! Key Exchange Algorithm KEA is a variation of the Diffie-Hellman algorithm and was proposed as a method for key exchange.  
124//! <https://www.fortinet.com/resources/cyberglossary/what-is-cryptography>

The GCM algorithm provides data authenticity, integrity and confidentiality and belongs to the class of authenticated encryption with associated data (AEAD) methods.

246/// Function main() returns anyhow::Result.
247fn main_returns_anyhow_result() -> anyhow::Result<()> {
248    tracing_init()?;
249    msg_enc_dec_config_initialize()?;
250    // super simple argument parsing. There are crates that can parse more complex arguments.
251    match std::env::args().nth(1).as_deref() {
252        None | Some("--help") | Some("-h") => print_help()?,
253        // Register completion for msg_enc_dec  with the shell command 'complete -C'.
254        Some("activate_completion") => activate_completion()?,
255        // When registered completion calls msg_enc_dec, the first argument is the path of the program.
256        Some("msg_enc_dec") => msg_enc_dec_completion()?,
257        Some("create_ssh_key") => create_ssh_key()?,
258        Some("send_public_key") => send_public_key()?,
259        Some("receive_public_key") => receive_public_key()?,
260        Some("message_encrypt") => message_encrypt()?,
261        Some("message_decrypt") => message_decrypt()?,
262
263        Some("file_encrypt") => match std::env::args().nth(2).as_deref() {
264            // second argument
265            Some(file_name) => {
266                file_encrypt(file_name)?;
267            }
268            None => eprintln!("{RED}Error: Missing arguments `file_name`.{RESET}"),
269        },
270        Some("file_decrypt") => match std::env::args().nth(2).as_deref() {
271            // second argument
272            Some(file_name) => {
273                file_decrypt(file_name)?;
274            }
275            None => eprintln!("{RED}Error: Missing arguments `file_name`.{RESET}"),
276        },
277
278        _ => eprintln!("{RED}Error: Unrecognized arguments. Try `msg_enc_dec --help`{RESET}"),
279    }
280    Ok(())
281}

420/// Print one or more sub_commands.
421pub fn completion_return_one_or_more_sub_commands(sub_commands: Vec<&str>, word_being_completed: &str) {
422    let mut sub_found = false;
423    for sub_command in sub_commands.iter() {

425            println!("{sub_command}");
426            sub_found = true;
427        }
428    }
429    if !sub_found {
430        // print all sub-commands
431        for sub_command in sub_commands.iter() {
432            println!("{sub_command}");
433        }
434    }

464/// Print the static public key to be sent.
465fn send_public_key() -> anyhow::Result<()> {
466    // https://docs.rs/crate/x25519-dalek/
467    // create static secret, because ephemeral secrets cannot be extracted and eaved to file.
468    let static_secret: x25519_dalek::StaticSecret = x25519_dalek::StaticSecret::random();
469
470    // Save the static secret encrypted into local folder.
471    let private_key_file_name = get_private_key_file_name()?;
472    let encrypted_secret_file_path = CrossPathBuf::new("static_secret.enc")?;
473    encrypt_and_save_file(&private_key_file_name, static_secret.to_bytes(), &encrypted_secret_file_path)?;
474
475    // Send the public key to the other party.
476    let public_key = x25519_dalek::PublicKey::from(&static_secret);
477    let public_key_string = ende::encode64_from_bytes_to_string(public_key.to_bytes().to_vec());
478
479    println!("  {YELLOW}Send this public key to the other party. This is not a secret. {RESET}");
480    println!("  {YELLOW}They must use 'msg_enc_dec receive_public_key'. {RESET}");
481    println!("  {YELLOW}and then send you the encrypted session token. {RESET}");
482    println!("  {YELLOW}It is encrypted, only the owner of the private key can decrypt it. {RESET}");
483    println!(r#"{GREEN}{public_key_string} {RESET}"#);
484
485    Ok(())
486}

488/// Save the secret bytes symmetrically encrypted into a file.
489///
490/// Use the private key to sign the random seed. The random seed is saved as plain inside the file.
491/// The file is bas64 only to masquerade it a little bit.
492fn encrypt_and_save_file(
493    private_key_file_name: &str,
494    secret_bytes: [u8; 32],
495    encrypted_secret_file_path: &CrossPathBuf,
496) -> Result<(), anyhow::Error> {
497    let (plain_seed_bytes_32bytes, plain_seed_string) = ende::random_seed_32bytes_and_string()?;
498    let private_key_path = CrossPathBuf::new(&format!("~/.ssh/{private_key_file_name}"))?;
499    let secret_passcode_32bytes: SecretBox<[u8; 32]> =
500        ende::sign_seed_with_ssh_agent_or_private_key_file(&private_key_path, plain_seed_bytes_32bytes)?;
501    let secret_string = secrecy::SecretString::from(ende::encode64_from_32bytes_to_string(secret_bytes)?);
502    let plain_encrypted_text = ende::encrypt_symmetric(secret_passcode_32bytes, secret_string)?;
503    let json_struct = EncryptedTextWithMetadata {
504        plain_seed_string,
505        plain_encrypted_text,
506    };
507    let json_string = serde_json::to_string_pretty(&json_struct)?;
508    encrypted_secret_file_path.write_str_to_file(&json_string)?;
509    Ok(())
510}

512/// Receive public key, calculate shared secret, encrypt and store for later use.
513fn receive_public_key() -> anyhow::Result<()> {
514    let other_public_key = inquire::Text::new(&format!("{BLUE}Copy the public key received from the other party:{RESET}")).prompt()?;
515    let other_public_key = ende::decode64_from_string_to_32bytes(&other_public_key)?;
516    let other_public_key = x25519_dalek::PublicKey::from(other_public_key);
517
518    // load and decrypt the static secret
519    let private_key_file_name = get_private_key_file_name()?;
520    let enc_file_path = CrossPathBuf::new("static_secret.enc")?;
521    let static_secret_bytes = load_and_decrypt(&private_key_file_name, &enc_file_path)?;
522    let static_secret = x25519_dalek::StaticSecret::from(static_secret_bytes);
523
524    // calculate shared secret
525    let shared_secret = static_secret.diffie_hellman(&other_public_key);
526
527    // save encrypted shared secret
528    let private_key_file_name = get_private_key_file_name()?;
529    let encrypted_secret_file_path = CrossPathBuf::new("shared_secret.enc")?;
530    encrypt_and_save_file(&private_key_file_name, shared_secret.to_bytes(), &encrypted_secret_file_path)?;
531
532    // for debugging I can write the encrypted session token that is created
533    // let session_token_enc_path = CrossPathBuf::new("enc_session_token_1.txt")?;
534    // session_token_enc_path.write_str_to_file(&plain_session_token)?;
535
536    println!("  {YELLOW}The shared secret session token is saved.{RESET}");
537    println!("  {YELLOW}Now you can encrypt and decrypt messages and files.{RESET}");
538    println!(r#"{GREEN}msg_enc_dec message_encrypt {RESET}"#);
539    println!(r#"{GREEN}msg_enc_dec message_decrypt {RESET}"#);
540    println!(r#"{GREEN}msg_enc_dec file_encrypt file_name {RESET}"#);
541    println!(r#"{GREEN}msg_enc_dec file_decrypt file_name {RESET}"#);
542
543    Ok(())
544}

546// Load and decrypt secret.
547fn load_and_decrypt(private_key_file_name: &str, encrypted_secret_file_path: &CrossPathBuf) -> Result<[u8; 32], anyhow::Error> {
548    let encrypted_secret_file_string = encrypted_secret_file_path.read_to_string()?;
549    let json_struct: EncryptedTextWithMetadata = serde_json::from_str(&encrypted_secret_file_string)?;
550    let plain_seed_bytes_32bytes = ende::decode64_from_string_to_32bytes(&json_struct.plain_seed_string)?;
551    let private_key_path = CrossPathBuf::new(&format!("~/.ssh/{private_key_file_name}"))?;
552    let secret_passcode_32bytes: SecretBox<[u8; 32]> =
553        ende::sign_seed_with_ssh_agent_or_private_key_file(&private_key_path, plain_seed_bytes_32bytes)?;
554    let secret_string = ende::decrypt_symmetric(secret_passcode_32bytes, json_struct.plain_encrypted_text)?;
555    let secret_bytes = ende::decode64_from_string_to_32bytes(secret_string.expose_secret())?;
556    Ok(secret_bytes)
557}

569fn message_encrypt() -> anyhow::Result<()> {
570    let private_key_file_name = get_private_key_file_name()?;

576    let shared_secret = SecretBox::new(Box::new(shared_secret_bytes));
577
578    let secret_message = inquire::Text::new(&format!("{BLUE}Write secret message to encrypt.{RESET}")).prompt()?;
579    let secret_message = SecretString::from(secret_message);
580    // encrypt secret message with symmetric encryption
581    let encrypted_message = ende::encrypt_symmetric(shared_secret, secret_message)?;
582    println!("Encrypted message:");
583    println!("{encrypted_message}");
584    Ok(())
